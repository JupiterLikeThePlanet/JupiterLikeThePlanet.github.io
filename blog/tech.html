Ruby on Rails

October 11th 2015

So what is this Rails I hear so much about?

So you know Ruby, the programming language? Yes, of course you do. Well, Rails is essentially a Ruby libray. Or, if you're a cool person, you may call it a gem. Rails extends the Ruby language and is a framework for building websites. It also:

  • combines Ruby with HTML,CSS, and good ol' Javascritpt to create web applications.
  • provides classes for saving to the database
  • handles URLs and displays html

So while Ruby is a language that can be used itself, Rails is a framework for web application written in Ruby. It provides a structure for your ruby code and includes some stuff that makes tasks much easier.

What makes Rails so popular?

It's code is seen as clean, eliminating what could take chunks in other languages. A developer ends up using less code and their applications will be similiarly structured. So, basically, it speeds up the routine blocks and makes work time more efficient. From reading around the block, developers say it allows them to spend more time with the problems at hand because the framework is already established. It's like taking a road trip, I don't have to build the car or the roads, I just jump in and go! Some specific reasons for it's use are:

  • Model View Controller: "break up your data and logic (the Model), what it looks like (the View), and how it interacts with the user (the Controller) into separate areas"
  • Ruby Gems: allows DLC to add to Rails
  • Expressiveness: it's quite an intelligeable langauge even to the non programmer (at least compared to other languages.
  • Dedicated Community: being that it is used for websites, it can be easily used in collaboration and maintenance.
